# Cognis 项目管理计划

## 1. 项目概况

### 1.1 项目目标
构建 macOS 平台上体验最好的 AI 原生终端模拟器，通过"双轨会话"技术实现人机无缝协作。

### 1.2 成功标准
- **技术指标**: SSH连接延迟<50ms，AI响应延迟<2s，内存占用<500MB
- **用户指标**: Beta用户留存率>40%，NPS>50
- **质量指标**: 崩溃率<0.1%，关键Bug修复时间<24小时

## 4. 风险管理

### 4.1 风险登记册
| ID | 风险描述 | 可能性 | 影响 | 缓解措施 | 负责人 |
|----|----------|--------|------|----------|--------|
| R1 | libssh2集成不稳定 | 中 | 高 | 尽早进行压力测试；考虑备选方案(BlueSocket) | Tech Lead |
| R2 | AI响应延迟过高 | 高 | 中 | 优化Prompt；使用流式响应；提供本地模型选项 | AI Eng |
| R3 | SwiftTerm性能瓶颈 | 低 | 高 | 参与开源社区优化；必要时自定义渲染层 | iOS Dev |
| R4 | App Store审核拒接 | 中 | 高 | 提前研究相关条款；准备申诉材料；提供非沙盒版本 | Tech Lead |

### 4.2 变更管理
- 所有需求变更必须经过Tech Lead评估
- 影响超过3人日的变更需调整排期
- 冻结期（Release前1周）原则上不接受新功能

## 5. 质量保证

### 5.1 测试策略
- **单元测试**: 覆盖所有Model和ViewModel逻辑
- **UI测试**: 覆盖关键用户路径（连接、执行命令、断开）
- **Dogfooding**: 开发团队内部全面使用Cognis替代现有终端工具

### 5.2 代码规范
- 遵循 Swift API Design Guidelines
- 使用 SwiftLint 强制格式
- 提交前必须通过 CI构建
